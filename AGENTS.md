# OFA (mysimtek) — Agent Guide

Flutter mobile app for ISP customers ("Pelanggan"), branded as **OFA**. Single-package, Android-only app (firebase_options.dart only defines Android). Backend is a separate Laravel project (`simtek-billing`).

## Commands

```bash
flutter pub get              # install dependencies
flutter run                  # run on connected device/emulator
flutter build appbundle --release  # release AAB
flutter build apk --release         # release APK
flutter analyze              # lint (uses flutter_lints defaults)
```

No tests exist in this repo. No CI/CD configured.

## Architecture

- **State management**: `Provider` + `ChangeNotifier`
- **API layer**: `lib/core/services/api_service.dart` — custom `http` wrapper, targets `https://mysimtek.coglinetech.com/api`
- **Auth**: Bearer token stored in `SharedPreferences` via `StorageService`. Login gated to `role == 'customer'`; other roles rejected with error message. `AuthProvider.checkAuth()` called on splash — if token exists, fetches `/me`.
- **Feature structure**: `lib/features/{auth,customer}/` — each has `screens/`, `providers/`, `models/`
- **Shared widgets**: `lib/core/widgets/` — `app_loading`, `app_error_view`, `app_empty_state`, `no_internet_overlay`, `app_webview`, `status_badge`

## Firebase / FCM

- Firebase project: `mysimtek-pelanggan`
- `lib/firebase_options.dart` is generated by FlutterFire CLI — **do not edit manually**
- FCM token synced to server on login and `checkAuth()` via `POST /api/fcm-token`
- Token cleared on logout (Firebase `deleteToken()` + server sync)
- Foreground data-only messages shown via `flutter_local_notifications` on channel `mysimtek_high_importance`
- Background handler is top-level function `_firebaseMessagingBackgroundHandler` (entry-point annotated)
- Notification taps navigate to billing/ticket/complaint detail screens based on `data['type']`

## Key conventions

- **UI language**: Indonesian — error strings, labels, comments all in Bahasa Indonesia
- **Theme**: Material 3, seed color `#FF6B00` (orange), Poppins font via `google_fonts`
- **API response format**: Laravel-style envelope `{ status, data, message? }` — `ApiService._parse` auto-unwraps `data` when no extra keys exist
- **App branding**: `lib/core/constants/app_version.dart` — `appName = 'OFA PELANGGAN'` (hardcoded)
- **FCM channel ID**: `ofa_high_importance` (AndroidManifest + FcmService)
- **App icon**: `flutter_launcher_icons` uses `assets/logo/ofa_logo.png`
- **WhatsApp admin numbers**: `lib/core/constants/whatsapp_admin.dart`
- **ProGuard**: enabled for release build (`minifyEnabled`, `shrinkResources`)

## Build / Release

- Signing config read from `android/key.properties` (gitignored)
- Keystore at `android/app/mysimtek-keystore.jks` (gitignored)
- `coreLibraryDesugaring` enabled for Java 8+ API support
- App icon generated via `flutter_launcher_icons` using `assets/logo/ofa_logo.png`
- Assets: `assets/icon/`, `assets/logo/`

## What NOT to do

- Do not manually edit `firebase_options.dart` or `google-services.json`
- Do not add tests without checking with user first (no testing infra exists)
- Do not change base URL in `api_constants.dart` without explicit instruction — production is `https://mysimtek.coglinetech.com/api`
